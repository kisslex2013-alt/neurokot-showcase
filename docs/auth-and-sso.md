# Auth / SSO (placeholder)

MVP может работать без авторизации (например, внутренний доступ через приватный URL).

Для полноценного внутреннего использования:
- Вариант 1: proxy-auth (внутренний reverse proxy, добавляющий заголовки пользователя)
- Вариант 2: OIDC/SAML (через корпоративный IdP)
- Вариант 3: Vercel + корпоративный SSO (если применимо)
